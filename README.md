# OULAD At-Risk Student Prediction

This repository contains a reproducible experiment pipeline for an educational data mining paper direction:

**Early prediction of at-risk students in online learning using multi-window behavioral features and explainable machine learning.**

The task is binary classification on the Open University Learning Analytics Dataset (OULAD):

- `at_risk = 1` for `Fail` or `Withdrawn`
- `at_risk = 0` for `Pass` or `Distinction`

## Dataset Preparation

Place the official OULAD CSV files in:

```bash
data/oulad/raw/
```

Expected files:

- `studentInfo.csv`
- `studentRegistration.csv`
- `studentAssessment.csv`
- `assessments.csv`
- `studentVle.csv`
- `vle.csv`
- `courses.csv`

If the files are missing, download the official Figshare dataset:

https://figshare.com/articles/dataset/OULAD_Open_University_Learning_Analytics_Dataset/5081998

Download `anonymisedData.zip`, extract it, and place the seven CSV files in `data/oulad/raw/`.

## Install Dependencies

Python 3.13 was used for this run.

```bash
python3 -m venv .venv
.venv/bin/python -m pip install --upgrade pip wheel setuptools
.venv/bin/python -m pip install -r requirements.txt
```

The pipeline uses optional model libraries when available: XGBoost, LightGBM, and CatBoost. If one is not importable, the corresponding model is skipped.

## Run the Complete Experiment

```bash
.venv/bin/python -m src.run_experiments
```

This single command:

- validates raw data files
- builds prediction-window feature matrices for 7, 14, 28, 56 days, and full course period
- trains Logistic Regression, Decision Tree, Random Forest, Linear SVM, XGBoost, LightGBM, CatBoost, and a stacking ensemble when available
- runs stratified student-group train-test evaluation
- runs GroupKFold by course presentation
- runs leave-one-course-presentation-out evaluation
- runs feature ablations
- generates SHAP explanations for the selected tree-based model
- writes all tables, figures, and reports under `outputs/`

To skip SHAP during debugging:

```bash
.venv/bin/python -m src.run_experiments --skip-shap
```

To reproduce the additional repeated-seed robustness and significance analysis:

```bash
.venv/bin/python -m src.repeated_significance --n-seeds 10
```

This command reruns the main 56-day model comparison over 10 student-grouped random splits, reruns XGBoost over all prediction windows, performs paired Wilcoxon signed-rank tests with Holm correction, and writes the repeated-experiment tables, figures, and report under `outputs/`.

## Leakage Controls

The pipeline is designed to avoid future-data leakage:

- `final_result` is used only to create the label and is not a feature.
- `id_student` is used only for joins and explanation identifiers, not as a model feature.
- `date_unregistration` is excluded because it directly reveals withdrawal.
- VLE features use only rows with `date <= prediction_day`.
- Assessment score features require both assessment due date and submission date to be on or before the prediction day.
- Missing due assessment counts include only assessments already due by the prediction day.
- The primary stratified train-test split is grouped by `id_student` so repeated students do not appear in both train and test.
- `days_before_start_registered` is only populated when the registration date is on or before the prediction day.
- Preprocessing is fitted inside sklearn pipelines after each train/test split or validation fold.

## Source Layout

- `src/config.py`: paths, constants, prediction windows, feature definitions
- `src/load_data.py`: raw CSV validation, loading, label creation, dataset summary
- `src/build_features.py`: demographic, registration, VLE, and assessment feature engineering
- `src/train_models.py`: preprocessing pipelines and model specifications
- `src/evaluate.py`: metrics and plots
- `src/run_experiments.py`: main orchestration entry point
- `src/repeated_significance.py`: repeated-seed experiments, paired significance tests, and related report generation
- `src/shap_analysis.py`: global and local SHAP explanations
- `src/leakage_audit.py`: leakage checks and Markdown report
- `src/utils.py`: shared helpers
